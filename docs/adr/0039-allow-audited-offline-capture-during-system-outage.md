# Allow audited offline capture during system outage

Child follow-up must not stop when the web app, Google Apps Script, or supporting Google Sheet is temporarily unavailable, but records captured outside the system must not silently become certified reporting data. During an outage, field workers may continue follow-up using a defined minimal fallback record through their service unit; when service resumes, unit staff will enter it through the web app as `บันทึกแทนจากเหตุระบบขัดข้อง`, retaining the actual follow-up date, actual field worker, entry timestamp, recorder, and outage reason. Eligible contact performance is attributed by the actual follow-up date, while official vaccine or registry-status changes still require existing verification; any fallback records not imported or verified by monthly close keep the unit visibly incomplete.
