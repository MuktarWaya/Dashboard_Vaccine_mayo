# Dry-run one unit before district-wide launch

The district already has legacy registers from all 14 service units and needs real-time district visibility from the launch day, so a prolonged phased rollout would not meet the operational objective. Before training and full import, the project will run a short technical dry run using real data from one unit to test import, validation, dashboard output, authentication, and mobile workflows and address known data issues, while preserving a district-wide launch for all 14 units on the actual go-live date.
