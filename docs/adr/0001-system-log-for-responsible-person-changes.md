# Keep responsible-person changes in a system log

The 14 service-unit registers must retain one shared structure while the dashboard must report historical responsibility accurately when an อสม. or ผรส. changes. We will keep only the current responsible persons in each service-unit row and have Google Apps Script write each actual change to a central `LOG_ผู้รับผิดชอบ` sheet with CID, service unit, responsibility type, old name, new name, effective month, editor, and edit timestamp, rather than asking field staff to add history columns or record monthly history manually.
